Que es git

¿Que es Git y para que sirve? - Segun Victor Robles Web

Git es un sistema de control de versiones. Un sistema de control de versiones nos va a servir para trabajar en equipo de una manera mucho más simple y optima cuando estamos desarrollando software.

Con Git vamos a poder controlar todos los cambios que se hacen en nuestra aplicación y en nuestro código y vamos a tener control absoluto de todo lo que pasa en el código, pudiendo volver atrás en el tiempo, pudiendo abrir diferentes ramas de desarrollo, etc.

Que es github

GitHub es un sistema de gestión de proyectos y control de versiones de código, así como una plataforma de red social diseñada para desarrolladores. ¿Pero para qué se usa GitHub? Bueno, en general, permite trabajar en colaboración con otras personas de todo el mundo, planificar proyectos y realizar un seguimiento del trabajo.

Pasos para usar git

Lo primero es tener una carpeta donde se almacenen los projectos, es decir que al menos contenga un html y un JS.

A esa carpeta le daremos un click derecho y seleccionamos Git bash here o abrimos CMD escribimos cd Desktop y entramos a la carpeta en caso de que esta esté en la pantalla principal.

Luego escribimos el comando git init. Esto también se puede hacer desde la consola de comandos de git.

Para ver los archivos con los que estamos trabajando usamos git status. Luego agregamos uno o varios archivos al statin area con el comando git add. Con eso los elementos de una carpeta serán agregados y se convertirán en la versión inicial de desarrollo.

Para crear un punto de partida, es decir un snapshot se usa el comando git commit. Luego pide que configuremos email y nombre de usuario. Este registro hay que hacerlo antes de el comando git commit.

Una vez que registremos un commit va a abir un editor de codigo en el terminal llamado bin donde se nos va a pedir un nombre para este, para salir de este registro una vez puesto un nombre es presionar esc y escribir :wq

Luego podemos ver este registro con el comando git log donde se muestran los commits creados con varios datos como un id unico para cada commit llamado hash.

Si hay una carpeta o archivos que no se quieren tomar en cuenta para el git lo que hay que hacer es crear un archivo .gitignore abrirlo y nombrar alli los archivos o carpetas que no son relevantes. Luego de eso solo se agrega el .gitignore a git.

Para que al hacer un snapshot no se abra el terminal, debe usarse el comando git commit -m "Comentario"

Existe el comando git branch para ver las ramificaciones, al principio solo habra una master que será la original, para crear una nueva se usa git branch nueva_rama, y para cambiar de rama se usa git checkout nueva_rama

Comandos basicos


pwd = Muestra tu locación actual

ls = Muestra las carpetas y archivos disponibles

git init = Inicia un nuevo proyecto

git status = Para ver el estado de nuestros archivos

git commit = Para crear un primer punto de control de nuestro código

git config --global user.email = Para configurar el email de este usuario

git config --global user.name = Para configurar el nombre de este usuario

git log = Para ver los commits creados

git checkout -- = Para revertir los cambios de los archivos

 git diff = Para ver las diferencias hechas en los archivos

git branch = Muestra las ramas que tiene un proyecto o crea una nueva

git add . = Agrega todos los archivos no incluidos

git clone = luego se escribe una url de github para descargar una carpeta contenedora de codigo y demás
